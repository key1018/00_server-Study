GET방식으로 요청 후 응답페이지 받아보기

특징1. get방식으로 요청시 url의 Header영역에 데이터를 포함시켜 요청
=> 사용자가 입력한 값(데이터)들의 url에 노출됨
=> 즉, 로그인이나 회원가입같은 경우 get방식에 부적합함

특징2. Header영역은 전송하는 길이에 제한이 있음
=> 방대한 데이터를 담았을 경우 초과된 데이터는 절단되어서 넘어감
=> 즉, 게시판 작성 같은 경우 get방식에 부적합함

특징3. 장점이라고 한다면 url에 데이터가 노출되어있기 때문에 즐겨찾기(북마크)를 통해 url을 저장해놨다가 재요청 가능
=> 즉, 검색 기능 같은 경우 get방식에 적합함
